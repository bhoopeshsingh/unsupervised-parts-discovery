Explanation Generation
experiments/run_pipeline.py  lines 381–530
▶
Activation Per-concept patch proportion for the query image (same as S5 formula)
Contribution contribution_i = coef_row[i] × (score_i − mean_score)
coef_row = LogReg weight row for the predicted class (not the true class)
mean_score = mean activation across all training images for that concept
Spatial map 28×28 grid coloured by cluster assignment per patch → overlaid on original image
Bar chart Signed contribution bars — positive = supports prediction, negative = suppresses
Bar length = contribution magnitude. Same concept can show different bar length for different class predictions because the weight row (coef_row) changes per predicted class.

src/models/lab_extractor.py  lines 62–146
▶
Method method='clinical_deviation' (config_lab.yaml line 151)
Formula deviation = (value − midpoint) / half_range
midpoint = (low_normal + high_normal) / 2
half_range = (high_normal − low_normal) / 2
Capped to [−4, +4] (line 146)
Example WBC range [4.5, 11.0] → midpoint=7.75, half_range=3.25
WBC=15.0 → deviation = (15−7.75)/3.25 = +2.23
Ranges 34 clinical ranges defined in config_lab.yaml lines 155–192. Z-score fallback for unmapped features.
Output cache/lab/features.pt  [N, 34] clinical deviation tensor
This is a rule-based normalisation, not learned. It encodes clinical meaning: +2.0 means "2 half-ranges above normal", which is medically interpretable.

Shared Architecture & Design Decisions

GMM vs K-Means

  • Both pipelines use GaussianMixture (sklearn), NOT K-Means at inference time
  • find_optimal_k.py uses MiniBatchKMeans only for the K-sweep (speed proxy)
  • K selected empirically: silhouette peaked K=10 on cat-only pilot (score=0.61)
  • GMM uses covariance_type="diag" — handles unequal cluster sizes
  • Image: 1 global GMM on all patches · Lab: 3 per-panel independent GMMs

No Label Leakage (Unsupervised Claim)

  • Image: class labels (cat/car/bird) only enter at S6 (LogReg training)
  • Lab: diagnosis labels (DIQ010, BPQ020) loaded after all L1–L6 stages complete
  • Concept naming (S4/L6) is post-hoc — names don't affect clustering or accuracy
  • Feature extraction is entirely self-supervised (DINO SSL / PSFTT SSL)

Ablation Results (Image)

  • A (Full): NMI 0.326, purity 0.891, silhouette 0.052
  • B (Single-layer L12): NMI 0.298, purity 0.875 (−0.028 NMI)
  • C (No FG mask): NMI 0.267 (−0.059 NMI vs full)
  • D (Pre-finetune): NMI 0.363, purity 0.935 — higher NMI because fine-tuning tightens semantic geometry at cost of class separation
  • Concept-based 98.88% vs raw baseline 46.42% (+52.46 pp)
